# `@tailwind-styled/dashboard`

HTTP server yang menyajikan build metrics `tailwind-styled-v4` secara live ‚Äî performa engine, history snapshot, dan health check.

---

## Quick Start

```bash
# Via CLI
tw dashboard

# Atau langsung
node -e "import('@tailwind-styled/dashboard').then(m => m.startDashboard())"
```

Buka `http://localhost:3000` di browser.

---

## Endpoints

| Method | Path | Response | Keterangan |
|---|---|---|---|
| `GET` | `/` | HTML | Dashboard UI |
| `GET` | `/metrics` | JSON | Snapshot metrics saat ini |
| `GET` | `/history` | JSON array | Semua snapshot metrics |
| `GET` | `/summary` | JSON | Agregasi history (avg, min, max) |
| `GET` | `/health` | `{ ok: true }` | Health check |
| `POST` | `/reset` | `{ ok: true }` | Reset history |

---

## Metrics Format

  "uniqueClasses": 891,
  "cssLength": 18540,
  "cacheHitRate": 0.97,
  "mode": "incremental",

## Cara Kerja

Server membaca metrics dari file `.tw-cache/metrics.json` yang ditulis oleh engine saat build. Perubahan file dipantau via `fs.watch` ‚Äî jika tidak tersedia, fallback ke polling setiap 2 detik.

```

                          ‚îÇ
Dashboard server ‚óÑ‚îÄ‚îÄ fs.watch ‚îÄ‚îÄ‚îÄ update in-memory state
        ‚îÇ

```

## File Utama

| File | Keterangan |
|---|---|
| `src/server.ts` | HTTP server ‚Äî routing semua endpoint |
| `src/server.mjs` | ESM wrapper untuk import langsung |
| `src/state.ts` | In-memory state ‚Äî `currentMetrics`, `history`, `updateMetrics` |
| `src/state.d.ts` | Type declarations state |
| `src/index.ts` | Entry point ‚Äî re-export `startDashboard` |

## Environment Variables

| Variable | Default | Keterangan |
|---|---|---|
| `PORT` | `3000` | Port HTTP server |
| `STUDIO_VERBOSE` | tidak ada | Tampilkan stdout/stderr server |

## Integrasi dengan CLI

Dashboard dijalankan via perintah `tw dashboard` di `@tailwind-styled/cli`. Studio Desktop juga me-forward metrics ke dashboard via IPC.
